Classes:

  • Classes are created using either the valor the var keywords. When created with val, the variable itself (the reference to the object) is read-only, however the object itself remains mutable.

When defining a class, you specify the properties and methods that all objects of that class should have.

class [name] { [body] }

Properties

  • define the class’ characteristics or data attributes, but are more flexible than variables through the use of getter and setter functionality
var name: DataType = initial_value
    get() {
        [body]
        return statement
    }
    set(value) {
        [body]
    }
  • When getter and setter functionality is not explicitly created, Kotlin creates these in the background as per basic function of a variable. Immutable val types will not have setter functionality.
  • Kotlin holds the value of variables in memory via a backing field, which is a class variable defined internally in the properties.
var namedVariable = 2
    get() = field
    set(value) {
        field = value
    }

Nullability:

  • A data type is only nullable if it is explicitly stated to be a nullable type
  • var [name]: [type]? = [value]
  • Null Safety:
    • Kotlin will not let you access properties of nullable variables to ensure no accidental calls are made on null objects.
    • While a variable may be null, when a member of a variable is accessed, the variable cannot be null as this would result in a null reference.
    • Kotlin forces a null check for all nullable types
  • Safe-Call Operator: ?.
    • used to access methods or properties of nullable variables
    • will return null if the variable is null
    • var name: String? = "SomeName"
    • nameLength = name?.length
  • Not-Null Assertion Operator: !!.
    • asserts that the variable is not-null
    • if the variable is null, will throw a NullPointerException
    • should only be used with appropriate exception handling or can cause runtime errors
  • If/Else null checks:
    • if (nullableVariable != null) {body1} else {body2}
    • after performing a null-check, methods of nullable variables can be accessed with the . operator and no longer require the ?. operator
  • Elvis Operator:
    • in isolation:
      • val result = nullableString ?: "default value"
    • in conjunction with safe-call operator:
      • val [name] = [nullable_variable] ?. [method/property] ?: [default_value]

Miscellaneous notes:

  • Trailing lambda syntax:
    • Kotlin allows this special syntax for passing functions as parameters to functions.
    • The function passed must be the last parameter.
    • This functions identically to the function being placed within {} as the last parameter inside the parentheses.
    • The name of the final parameter need not be specified as it is inferred by the trailing lambda
  • Due to type inference in Kotlin and the fact that floating-point numbers are suffixed with a “f”, all decimal point numbers do not need their type declared and are inferred to be Double.
  • Unit in Kotlin has the equivalency of void in Java
  • As opposed to Java, in Kotlin the values of arguments passed as parameters to a function are immutable
  • Default & Named arguments: while function parameters can specify default arguments, when an argument is passed to the function, Kotlin will assume that the order of the arguments is the same order in which the parameters are defined. It is therefore preferable to use named arguments and / or define default parameters AFTER required parameters.
  • when takes the place of switch / when statements in Kotlin:
    when (parameter) {
    condition1, condition2, condition3, condition4 -> outcome1
		in [range_start]..[range_end] -> outcome2
    condition5 -> outcome3
		is [data_type] -> outcome4
    else -> default outcome
}
    • can be helpful to use this syntax with the data type: Any
      • val x: Any = 4
  • if / else expressions:
    • val [name] = if (condition) {value_1} else {value_2}
    • remember that expressions must always return a value
  • Kotlin treats functions as first-class constructs:
    • functions are as much a data type as an Integer or a String
    • Functions can be used as parameters and returned from functions
  • Lambda Functions:
    • shorthand syntax to declare a function
    • fun printHello(){ println("Hello") }
    • can be:
      • { println("Hello") }
      • or:
        • val sayHello = { println("Hello" }
  • OOP Fundamentals:
    • Encapsulation
      • Breaking down the aspects of a program into small, manageable chunks, contained within an object that contains all of the data and functions required to operate.
    • Abstraction
      • Including information hiding and an extension of encapsulation, this allows objects to be interacted with at a conceptual level, without knowledge of how the object is performing its functions.
    • Inheritance
      • Allows the creation of new classes based on existing ones, inheriting, extending and/or adjusting the attributes and behaviours of the parent class.
    • Polymorphism
      • The ability to use a single object in multiple ways via runtime/dynamic method overriding where a class provides its own implementation of a function or by compile-time/static overloading where multiple methods can have the same name but differing parameter signatures
